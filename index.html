Courses & Learning

Programming Basics & Dynamic Memory

This section introduced me to fundamental programming concepts such as primitive data types, variables, control structures, and functions. I also learned about pointers and dynamic memory allocation, which are essential in C programming. Understanding how to manage memory efficiently can lead to significant improvements in application performance, especially in environments with limited resources.

Big O Notation and Complexity

Learning algorithmic complexity taught me the importance of efficiency. Big O notation helps in estimating the performance of an algorithm relative to the size of the input. It provides a high-level understanding of how algorithms scale and allows for comparing different algorithms based on their time and space requirements.

List Structures (Linked Lists)

In this section, I learned about linked lists, a fundamental data structure that consists of nodes connected through pointers. Unlike arrays, linked lists allow for dynamic memory allocation and efficient insertion and deletion operations.

Stacks and Queues

Stacks and queues are essential data structures that represent collections of elements with specific access rules. Stacks follow the Last In, First Out (LIFO) principle, while queues operate on a First In, First Out (FIFO) basis.

Trees and Hashing

Trees are hierarchical data structures that allow for efficient searching, insertion, and deletion operations. Hashing provides a way to efficiently map keys to values, enabling constant time complexity for lookups in average cases.

DSA Problem-Solving Workflow

  • Problem Analysis: Read and analyze the problem statement, identify input/output requirements, and constraints.
  • Choosing Data Structures: Select the most efficient data structure for the problem (e.g., arrays, stacks, graphs).
  • Algorithm Design: Design an algorithm based on patterns, previous experience, or standard approaches (e.g., divide-and-conquer).
  • Implementation: Code the solution with an emphasis on readability, modularity, and test cases.
  • Optimization & Testing: Optimize the solution for time and space complexity and test it against edge cases.
